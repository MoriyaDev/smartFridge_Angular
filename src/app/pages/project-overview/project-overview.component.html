🚀 יצירת הפרויקט

בפרויקט שלי בניתי קומפוננטות עיקריות הכוללות את דף הבית, פתקים, מתכונים, מוצרים ומשתמשים. חילקתי את הקוד למודולים שונים, כך שהפרויקט מאורגן וקל לתחזוקה.

🗺️ ניתובים וטעינה עצלה

השתמשתי בטעינה עצלה (Lazy Loading) כדי לשפר ביצועים ולאפשר טעינה מהירה יותר של האתר. בנוסף, יצרתי נתיבים ייעודיים לכל דף בפרויקט.

📊 עבודה עם נתונים

בפרויקט נעשה שימוש בServices לניהול מוצרים, מקרר ופתקים, עם קריאות API (GET/POST) לשרת לקבלת ועדכון נתונים.

🛠️ שימוש ב-Pipe ו-Directive

יצרתי Pipe אישי כדי לבדוק אם אני נמצאת במקרר או במקפיא, וגם לחיתוך טקסטים ארוכים. בנוסף, השתמשתי בDirective המעניק צבע שונה לכל סוג פתק, מה שמסייע בזיהוי מהיר של הפתקים.

🔒 אבטחה והרשאות

הגדרתי Guards להגבלת גישה לדפים רק למשתמשים מורשים, והשתמשתי בJWT לאימות משתמשים, עם רמות הרשאה שונות למנהלים ולמשתמשים רגילים.

🔄 אינטרפטור להוספת טוקן

בכל קריאה לשרת, אינטרפטור (Interceptor) מוסיף אוטומטית את הטוקן (Token) לאימות המשתמש, כדי להבטיח אבטחת מידע.

🍽️ אינטגרציה עם API חיצוני

חיברתי את הפרויקט לAPI חיצוני שמספק מתכונים על בסיס המוצרים שבמקרר שלי. מכיוון שהמתכונים מגיעים באנגלית, שילבתי API נוסף של Google Translate כדי לתרגם אותם לעברית.

✅ ולידציות ובדיקות טפסים

כל הטפסים בפרויקט נבדקים עם ולידציות, כדי לוודא שהמשתמש מזין נתונים תקינים, וכדי למנוע שגיאות.

📡 תקשורת בין קומפוננטות

השתמשתי בInput/Output כדי להעביר מידע בין קומפוננטות שונות, כך שהמערכת פועלת בצורה דינמית ומגיבה לשינויים.
